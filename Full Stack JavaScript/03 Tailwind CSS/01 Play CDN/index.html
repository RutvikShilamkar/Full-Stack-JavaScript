Use the Play CDN to try Tailwind right in the browser without any build step. The Play CDN is designed for development purposes only, and is not the best choice for production.

  1. -see classes applied on html elements
  2. -use quick search option on tailwind website to see if classes exist or not. for example if you want margin and my-12 class. Search margin to confirm class my-12 exist or not.

  3. ##For production use PostCSS plugin and not PlayCDN
  4. - do plugin setup as per Tailwind website
  5. Steps & commands
  6. 1) Install node js first & restart PC
  7. 2) Open terminal in vs code and type command:
  8. "npm init"
    9. The npm init command is a step-by-step tool to scaffold out your project. It will prompt you for input for a few aspects of the project
  9. or npm init -y (y means yes to all)
  10. Is this OK? (yes) : type yes
  11. 3) We want to use Tailwind CSS in the production of our application, therefore we would install the Tailwind with the help of Post CSS, Vite, and Autoprefixer. To install Tailwind for production, simply execute the below command in the Terminal-
  12. "npm install -D tailwindcss postcss autoprefixer"
  13. 4)Installing Vite: After executing the above command, Install Vite with the help of the below command. In package.json, make sure to add a script section to run vite whenever npm start is executed.
  14. "npx tailwindcss vite"
  15. 5) Initialising TailwindFinally, for the initialization of the Tailwind, you have to use the below command
  16. "npx tailwindcss init"
  17. After the execution of the above command, a Tailwind.config.js file will be created. Make sure to add a “*” in the content section as we will be using the Tailwind in the complete content.
  18. 6) "npm run start"
  19. click on localhost link to open website
